文章正文

这篇文章主要为大家详细介绍了MySQL解决乱码办法，具有一定的参考价值，可以用来参考一下。

MySQL支持几十种编码方式，并且默认的编码为：latinl,如果MySQL出现了乱码情况，不要慌乱，一下为你介绍两种解决编码方式：

在控制台输入命令，注意MySQL中的UTF-8写成utf8，例如我现在有一个数据库名称为：lcychina，设置如下：


---- 来自www.512pic.com  
mysql> alter database lcychina character set utf8;

Query OK, 1 row affected

mysql>

查看当前数据库编码方式命令为：


---- 来自www.512pic.com  
mysql> show variables like 'character_set_database';

+------------------------+-------+

| Variable_name | Value |

+------------------------+-------+

| character_set_database | utf8 |

+------------------------+-------+

1 row in set

修改my.ini配置文件，用记事本打开，找到MySQL安装目录，我的MySQL安装的路径是：D:\Java\MySQL\MySQL Server 5.5

大概在57行的样子找到：

default-character-set=latinl

修改为：

default-character-set=utf8

应该有两行这样的代码，注意修改参数只会对以后建数据库或表才有影响。不会影响已经存在的数据库和表。
