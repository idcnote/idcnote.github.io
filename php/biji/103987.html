如何解决 OS X 系统安装或更新中 “安装过程已破坏”或“程序副本不能验证“  等问题“OS X Yosemite 应用程序副本不能验证”，或者“在下载过程中可能已遭破坏或篡改 ”等问题经常会在 OS X Yosemite 的安装过程中出现。有时候简简单重启或者重新安装就能解决问题。但是，如果该问题一直反复出现，那么可以通过修改 Mac 系统日期来解决问题。

1、基于 OS X Yosemite 系统的任何程序安装都有可能出现类似问题，可能会是 Apple Store 更新，网络恢复，程序卸载等。  如果用户在 Apple Store 进行 OS X 的常规安装或者更新遇到该问题，可以通过点击进入苹果菜单>系统偏好设置>日期&时间，选择自动调整日期和时间来解决问题。操作时得保证有网络访问。如果没有网络，就到终端进行日期设置。

2、为了确定终端日期设置是否解决了该问题重新安装 OS X，用户需在“安装OS X”启动菜单>命令行，下拉“实用工具’菜单选项，选择’终端”，然后输入以下命令：date点击回车键，如果报告日期是当前日期，那么就能找到该问题的原因了，如图所示：  Mon Jan 19 09:55:15 PST 1984 这一行内容非常重要，以为只有日期正确了才能成功安装 OS X，如果改日期遭遇 OS X 的发布时期，那么就会出现上述的错误，所以年份特为重要。如果时间错误，有两种方法进行设置，可以设置为自动调整日期，通过网络设置要简单方便一些： ntpdate -u time.apple.com 如果 Mac 没有网络访问就不能进行这项设置，因此，用户需要使用手动设置。 如果用户熟悉命令行则可使用 日期-帮助 进行检索，这种日期格式开起来可能会有点怪—“[mm]dd]HH]MM[[cc]yy]”，其实也就是“月日时分年”，这是一行固定格式，在数字之间不 能加入空格和其他符号，比如将日期设置为：“September 20 2016 at 17:33′′，用户可看到以下语法： date 0920173316 这看起来像是一组随机的数字，但其实就是月份09，时间1733，年份16，几个时间组成的。

3、一旦输入日期点击回车键，就能退出终端，开始 OS X 的安装了，并且再也不会出现上述的问题。
