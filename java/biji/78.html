在Windows 7下配置Java运行环境（使用JDK 1.6）

内容摘要
Win7旨在让人们的日常电脑操作更加简单和快捷，为人们提供高效易行的工作环境。
文章正文

　　Win7旨在让人们的日常电脑操作更加简单和快捷，为人们提供高效易行的工作环境。然而在运行java程序时却出现了各种各样的问题，接下来作者教你如何在WIN7下配置JDK1.6，并成功运行java程序。

　　WIN7下配置JDK1.6，成功编辑运行JAVA程序的过程：

　　1.安装的是jdk-6u22-windows-i586，安装路径为D:\Java\;

　　2.环境变量的配置：

　　(1)JAVA_HOME=D:\Java\jdk1.6.0_22;(路径各人按照各自的情况改就可以了)

　　(2)PATH=;%JAVA_HOME%\bin;(一定要在系统原有的PATH路径的后面添加，而不是直接覆盖，否则可能会出现无法使用CMD的情况。比如

　　原来的PATH的变量为C:\Windows\system32时，就在“……32”后边加英文;“%JAVA_HOME%\bin;”，如果“……32”后边有“;”

　　那就直接加上“%JAVA_HOME%\bin;”就行了。顺便说一下，出现CMD命令无法使用时，按照本方法，加上C:\Windows\system32就可以了)

　　(3)classpath=.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;(需要强调的是，1。变量前面的英文“.;”最好都加上，以防未知的错误发生;2。将变量改为“.;%JAVA_HOME%\jre\lib\rt.jar;%JAVA_HOME%\lib\tools.jar”，仍然可以照常使用，原因对于我这个菜鸟来说，尚且未知，望JAVA前辈指教。)

　　至此，可以运行java-version来查看版本，如果出现类似于javaversion"1.6.0.22"的字样，则表示安装成功。

　　3。编译入门级java程序，即无人不知无人不晓的HelloJava.java

　　3.1.根据各自的情况，如我，即使在E盘新建一个JAVAWORK文件夹，在其中新建记事本文件，然后修改格式为java，系统提示就确认;

　　3.2.用记事本打开，编写(或者直接考前提是明白其中的意义，这才是最重要的)以下程序，保存：

　　publicclassHelloJava
　　{
　　publicstaticvoidmain(Stringargs[])
　　{
　　System.out.println("哈哈哈哈!终于成功了!");
　　}
　　}

　　3.3.运行cmd，键入"javace:\JAVAWORK\HelloJava"如果没有什么提示，就说明创建类文件成功，回到该目录中就会看到HelloJava.class文件建成;

　　3.4.紧接着就运行javaHelloJava……

　　看到提示了吗?这说明说明呢?说明，java找不到我们创建的类文件。为什么你?是环境变量没有创建好么?就是那个classpath?

　　也对，也不对。首先，我们的类文件是在一个与JDK的系统文件不在一个硬盘或者文件夹中，也就是路径不同了;其次，直接用java调用类文件，java并不能根据我们设置的classpath找到这个名叫HelloJava的家伙。那么，如果我们将名叫HelloJava.class的家伙，放到classpath指定的路径里，是不是就可以了呢?我们汇过去看一下变量，都是指定的具体的java工具集和库文件，实打实的具体的文件，怎么办?硬是放进lib中，试试……还是不行吧?那么，能不能直接在classpath中，指定现在我们这位名叫HelloJava.class的老大的文件夹呢?按照上边的方法，在路径后边加上";E:\JAVAWORK\;"，退出CMD，刷新，再次运行javaHelloJava

　　4。对于菜鸟如我者来说，入门很重要，菜鸟我深知道。经过了一天的摸索，我搞清楚了这个一直存在的问题。希望能够帮助和我一样的菜鸟们。幸福的家庭彼此相似，不幸的家庭各有各的不幸。我想说，幸福的感觉彼此相似，困难的你我各有自己的困难，只要我们有决心，没有解决不了的问题的。

　　5。本机的操作系统：WIN7专业版，JDK版本：jdk-6u22-windows-i586。很多人可能觉得WIN7使用起来会有困难而望而却步，止步不前。
