甲
文章正文

Java SE 8预计在2013年8月出炉。Java SE 8的功能还包括改善JavaScript于JVM上的运作，采用新一代的JavaFX 3.0客户端程序，并支持更多的现代装置等。

甲骨文于本周举行的JavaOne会议中公布了目前各种Java规格与技术的进度，包括展示Java SE8的发展蓝图，阐述对Java EE7的规画，并推出JavaFX 2.0，以及宣布增加对Java ME的投资。

Java SE为标准版的Java平台（Java Platform Standard Edition），甲骨文为了要在今年8月如期推出Java SE7，将许多重大功能延至Java SE 8，诸如名为Jigsaw项目的模块化系统，可用来简化应用程序的建构、包装与部署，以及更能利用多核CPU与大量数据处理强化能力以提高生产力的 Lambda项目。

Java SE 8的功能还包括改善JavaScript于JVM上的运作，采用新一代的JavaFX 3.0客户端程序，并支持更多的现代装置等。Java SE 8预计在2013年8月出炉。

至于企业版的Java EE（Java Platform Enterprise Edition）6自2009年12月推出后，已有6家主要的IT业者发表经认证的开放源码或商业建置，使其成为迄今施行最快的版本。

尚在审核中的Java EE7则强化其云端环境能力，使该平台上的应用程序与产品更容易于私有或公共云端上操作，且将支持多重租赁与服务扩充弹性。

刚问世的JavaFX 2.0为一Java用户接口平台，甲骨文计划释出JavaFX平台的开放原始码，初期将释出UI控制器与相关的函式库，其他的组件亦会分阶段释出。支持Mac OS X的JavaFX 2.0公开测试版也在本周开放下载。

甲骨文亦宣布将加码对Java Platform Micro Edition（Java ME）行动平台的投资，预计在未来几个月会提出Java ME 7的规格需求以供与Java社群程序（Java Community Process）共同进行开发。

甲骨文表示，Java ME 7的规格将带来新的加值服务，以于功能性手机上提供智能型手机的经验。此外，根据估计，去年全球出货的Java Card达20亿，甲骨文期望新的Java Card 3.0.4平台将提振Java Card于NFC行动付款与M2M（machine to machine）等安全装置的采用。

依照TIOBE指数，Java现为全球最受欢迎的程序语言，全球有900万名Java开发人员，每年的Java下载量超过10亿，全球有超过 1.25亿台 以Java为基础的电视装置，有超过30亿台装置采用Java技术，此外，欧洲、美洲与日本的SIM卡有逾9成为Java Card。
