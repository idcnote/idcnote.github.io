四种方法提高Windows Server 2008下载速度_操作系统

内容摘要
相信不少朋友都会认为既然Windows Server 2008系统的功能十分强大，那么在该系统环境下进行信息下载操作时，速度也理应比传统操作系统下的下载速度
文章正文

　　相信不少朋友都会认为既然Windows Server 2008系统的功能十分强大，那么在该系统环境下进行信息下载操作时，速度也理应比传统操作系统下的下载速度更快一些。不过，当我们在Windows Server 2008系统环境下实际尝试下载操作时，感觉到在该系统环境下的信息下载速度，有时还不如以前那样迅速呢，这是什么原因呢？出现这种原因，主要是 Windows Server 2008系统在安全方面进行了强化，这给信息下载操作带来了不小的限制，从而制约了信息下载速度！为了提高Windows Server 2008系统下的信息下载速度，本文以迅雷下载工具为例，来向各位朋友推荐一些设置Windows Server 2008系统的技巧，确保Windows系统不会制约迅雷下载工具的信息传输速度！

　　1、 借助工具优化TCP/IP连接数

　　大家知道，在传统操作系统环境下，我们可以利用迅雷下载工具自带的系统优化功能，来对计算机的TCP/IP连接数进行优化，让系统缺省的TCP/IP连接数从"10"修改成合适的数值，这样就能保证迅雷下载工具同时利用多个线程进行极速下载了。不过，在Windows Server 2008系统环境下，当我们再尝试按照相同的方法来修改系统TCP/IP连接数时，发现迅雷下载工具自带的系统优化功能已经失效了，这么说来我们难道就没有办法对Windows Server 2008系统默认的TCP/IP连接数进行修改了？

　　其实，为了增加Windows Server 2008系统下的TCP/IP连接数，网上现在专门有修改TCP/IP连接数的工具程序可以下载，将该工具程序下载保存到Windows Server 2008系统中，然后借助专业工具解压，再执行解压包中的"install.bat"批处理文件，随后系统工作状态将会被自动切换到MS-DOS命令行状态，紧接着按下键盘中的"A"按键，选中默认安装方式安装该破解系统连接数的小工具，待安装操作结束后，重新启动一下Windows Server 2008系统，那样一来TCP/IP连接数就会被自动增加到合适的数值；此时，在Windows Server 2008系统环境下，我们就可以启用迅雷下载工具的多线程功能，来实现快速下载目的了。

　 2、启用系统特有自动调谐功能

　与传统操作系统相比，Windows Server 2008系统自带了TCP/IP自动调谐功能，该功能可以依照实际的网络工作环境，来对网络连接数值进行自动调节，确保网络始终能以理想的速度进行传输。不过，在实际上网冲浪的过程中，Windows Server 2008系统并不能很好地根据网络环境来调整上网连接数；有鉴于此，我们必须采用手工方法来启用Windows Server 2008系统自带的TCP/IP自动调谐功能，以便希望该功能能够优化迅雷下载工具的下载速度，下面就是具体的启用步骤：

　 首先以系统管理员权限登录进入Windows Server 2008系统，在该系统桌面中依次点选"开始"/"运行"命令，在弹出的系统运行文本框中，输入字符串命令"cmd"，单击"确定"按钮后，将系统工作状态切换到DOS命令行工作状态；

　其次在DOS命令行提示符下，输入字符串命令"netsh int tcp set global autotuninglevel=normal"，单击回车键后，系统屏幕上将会返回如图1所示的结果界面，这说明Windows Server 2008系统自带的TCP/IP自动调谐功能已经被成功启用了。此时，迅雷下载工具在Windows Server 2008系统环境下的下载速度就能得到明显提升了。

　 3、UPnP功能让下载速度再快一些

　 迅雷下载工具支持BT下载功能，而很多时候我们都是在局域网网络中进行BT下载操作，这时如果将Windows Server 2008系统的UPnP功能启用起来，就可以让下载速度再快一些。在启用Windows Server 2008系统的UPnP功能时，我们可以按照下面的操作来进行：

　 首先在Windows Server 2008系统桌面中依次单击"开始"/"程序"/"管理工具"/"服务器管理器"命令，打开对应系统的服务器管理器控制台窗口；

　 其次在该控制台窗口的左侧显示区域中，将鼠标定位于"配置"分支选项上，在对应"配置"分支选项的右侧显示区域中，双击"服务"选项，打开Windows Server 2008系统的服务列表窗口；

　 从该服务列表窗口中找到"SSDP Discovery"服务选项，用鼠标右键单击该目标服务选项，并从右键菜单中执行"属性"命令，打开"SSDP Discovery"服务的属性设置窗口；

　单击该设置窗口的"常规"标签，打开如图2所示的标签设置页面，在该页面中我们能够非常清楚地看到"SSDP Discovery"服务此刻的运行状态，要是发现目标系统服务已经被关闭运行时，我们可以直接单击对应设置页面中的"启用"按钮，来将"SSDP Discovery"服务先启用成功，之后再将该服务的启动类型参数调整为"自动"，最后单击"确定"按钮执行保存操作，如此一来Windows Server 2008系统就能支持UPnP功能了。

　在启用了Windows Server 2008系统的UPnP功能后，我们还需要打开迅雷下载工具的程序设置界面，将其中的"允许使用UPnP自动端口映射功能"选项以及"允许连接DHT网络"选项选中，这样一来我们日后在Windows Server 2008系统环境下通过迅雷下载工具从Internet网络中下载信息时，速度就能大大提升了。

　 当然，如果局域网中使用的路由器还没有启用UPnP功能时，我们还需要进入路由器的后台管理界面，从中找到"路由"标签设置页面，打开路由器参数设置页面，之后单击"转发"选项卡，并在对应选项设置页面中找到UPnP功能选项，检查UPnP功能目前是否已经处于运行状态，要是发现该功能还没有被启用时，那么我们只要单击对应设置页面中的" 启用UPnP"按钮，最后执行保存操作就可以了。

　　4、禁止模块安装拖累下载速度

　　在Windows Server 2008系统环境下，当我们利用迅雷下载工具下载一些大容量的多媒体文件时，时常会碰到系统CPU资源突然被100%消耗的现象，这种现象会直接拖累迅雷工具的下载速度。引起这种现象的原因，主要是Windows Server 2008系统的模块安装功能在捣乱；要是在下载信息的过程中，Windows Server 2008系统自动执行模块安装操作时，我们唯一能做的就是等待该安装操作结束，系统CPU资源恢复正常，才能继续进行信息下载操作。为了不让模块安装操作拖累信息下载速度，我们可以按照下面的设置来禁止模块安装操作拖累下载速度：

　　首先在Windows Server 2008系统环境下打开"开始"菜单，从中点选"运行"命令，打开对应系统的运行文本框，在其中输入字符串命令"services.msc"，单击"确定"按钮后，进入对应服务器系统的服务列表界面；

　　其次用鼠标双击服务列表界面中的"Windows Module Installer"选项，打开如图3所示的属性设置窗口；单击其中的"常规"选项卡，在对应的选项设置页面中单击"停止"按钮，将处于运行状态的"Windows Module Installer"服务关闭运行；

　　为了避免"Windows Module Installer"服务日后跟随服务器系统自启动运行，我们还必须在"常规"设置页面中将目标服务的"启动类型"参数修改为"手工启动"，那样的话我们日后使用迅雷下载工具下载大容量的多媒体信息时，就不会碰到系统CPU资源突然被100%消耗的现象了。
