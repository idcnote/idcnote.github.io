Longhorn 最新信息

内容摘要

昨天作了更新，内容有些变更，更加详细的介绍了Longhorn的个方面的信息。相信有许多你仍然不了解的问题。主要从Longhorn的开发路线图、开发计划，独特功能
文章正文

昨天作了更新，内容有些变更，更加详细的介绍了Longhorn的个方面的信息。相信有许多你仍然不了解的问题。主要从Longhorn的开发路线图、开发计划，独特功能，安装硬件要求等方面进行介绍。有些翻译不是太贴切，部分词语没有翻译，如有不明之处，请结合原文阅读。 

1、Longhorn路线图 

2005  
Windows XP Professional x64 Edition: April 2005  
Longhorn Client Beta 1: H1 2005  
Longhorn Client Beta 2: H2 2005 

2006  
Longhorn Client RTM: H1 2006  
WinFS Beta: H1 2006  
Virtual PC 2006: H1 2006 

2007  
Longhorn SP1: H1 2007  
Longhorn Server: H1 2007 

2、Longhorn 发布计划 

Longhorn Beta 1 Milestone 9 (Beta 1 M9) and platform code complete*——March 16, 2005 
Longhorn Beta 1 ——May 25, 2005 
Longhorn Beta 2 ——October 12, 2005 
Longhorn Client Release Candidate 0 (RC0) ——February 22, 2006 
Longhorn Client Release Candidate 1 (RC1) April 2006  
Longhorn Client release to manufacturing (RTM) ——May 24, 2006 
Longhorn Launch (widespread public availability) ——Summer 2006 to October 2006, depending on release schedules for other Longhorn wave products. 
Longhorn Server RTM/Longhorn Client SP1 RTM ——Second half of 2006/first half of 2007 (Client RTM + 6 months) 

3、强大、可靠、安全 

微软希望Longhorn能成为高效、坚固和安全的操作系统，为完成此目标，肯定需要彻底检查Windows帐户的工作情况。现在的XP已经支持限制帐户、管理员帐户，但是，人们一般都使用管理员帐户而很少问津限制帐户（图方便啊），所以限制帐户几乎成了摆设。 

那么，在Longhorn里，微软会推出新的最小权限帐户（LUA）。此帐户基本就是一个安全代码运行隔离区，大多数的应用程序只是象征性的运行（most application code will typically run）。如果某些程序需要管理员权限，其就会运行于受保护的管理员模式下。这个特性将可以避免许多家庭用户所面临的限制帐户问题，而商业用户管理员可以关闭它。 

与Windows XP SP2一样，Longhorn也拥有遇到错误操作发出警告并指导解决问题的向导。但是Longhorn可以像本地运行一样显示错误提示，而不像Windows XP SP2那样是基于网络的通讯。这对于用户来说会觉得更加安全，并且可以退回到以前的任何步骤，安全感加强了。 

总之，同Windows XP SP2相比Longhorn的安全性和管理性有了显著的提升。例如，XP SP2中新的安全策略在Longhorn中将显著得以扩展，尽管如此，其工作方式不变。所以，对于管理员来说将需要简短的培训学习即可了解组策略在Longhorn中是如何工作的。 

Longhorn中将支持一种新型的更新模式——Hot Patching，通过它将可以应用更新至任何非内核程序，包括驱动程序，不需要重起哦！当然，对于某些补丁Longhorn在更新后仍然需要重起，估计比XP SP2和Server 2003少70%的重起次数。 

另外，Longhorn具有instant-on（即时待命？）能力，可以在不到2秒内从等待状态进入工作。微软宣称，Longhorn的冷启动时间将比XP及其同等操作系统节约50%！ 

4、IT执行效率 

对于要将Longhorn部署到桌面的商业用户，同XP相比Longhorn将提供许许多多的优势，削减部署、管理及技术支持方面的投资。 

借助微软下一代的AntiSpyware（反间谍软件，目前处于1.0的测试期）及正在开发中的反病毒软件，Longhorn将具有探测间谍软件和恶意程序的能力，管理员可以扫描到PC及网络上的威胁。 

Longhorn还包括新型的EFS（Encrypting File System，文件加密系统）技术，可以阻止数据丢失或偷窃。 

Longhorn中应用程序的加载将比Windows XP中快15%。Longhorn会具有新的图像生成及管理工具。 

5、更加智能的工作方式 

Longhorn将提供更多的存取、管理及使用信息的途径，并大大提升了IW（Information Worker）的工作效率。其实现的关键问题是要有新型快速的搜索功能，可能许多人认为WinFS将会从Longhorn中去除，实际上不会的。Longhorn的快速搜索速度近乎瞬时，不论是本地计算机还是局域网、互联网，并且是原生于Windows的。 

快速搜索一新的方式组织数据，包括列表、自动列表和过滤器，如图1。搜索结果会包含不同类型的数据，如文档、邮件和图片等等，可以以自定义形式排列。Longhorn的新窗口也将具有便捷的"breadcrumb"向导功能，并包含基于快速搜索技术的过滤视图和文件预览。如图2。 

另一项快速搜索功能叫Stacks（堆叠，堆砌？），可以以诸如作者、关键字、类型等方式堆叠目录显示。要显示多个相关信息，也可以按不同属性分组Stacks，如：名称、大小、修改日期、类型或作者等。如图3。 

微软相信，快速搜索会节约用户80%的时间用于查找文件，改变文件同步的时间将比Windows XP快50%。 

说道WinFS，我们停一下。我们知道微软将在2006年中发布Longhorn时会发布一个相关的存储技术预览，届时WinFS将开启比快速搜索还要强大的搜索，微软甚至还没有决定到时候是否发布WinFS或如何包装和发布这项技术！ 

Longhorn将会更加可靠的从崩溃后恢复，包括更好的应用程序管理和备份及恢复功能。 

借助Longhorn可以轻易、无缝的创建基于P2P技术的专用网络，由此为多人协作铺通了宽阔大道。这样一个新的类域的网络方案将会取代工作组，在这种名为castle（城堡?）里的方案里，用户凭证可以从一台机器进入另一台机器而不需要中央服务器。

6、随时在线 

Longhorn中，计算机与移动设备间同步数据的能力将无比强大，在无线连接方面也是无比强劲。或者像微软所描述的那样：Longhorn将让你"work together and accomplish more anytime, anywhere." There you go.（无论所处何时何地，协同工作、做得更多？） 

Alpha版Longhorn就已经暗示Longhorn即将带来什么：一个通用的同步管理器将管理着软件和硬件间的连接。新无线网络将支持任何地方的远程访问和更多网络间的无缝转换，在有多个网络可选时会自动连接最大带宽的那个。 

7、下一代平台 

我们知道，Longhorn基于三个重要的部分：图形子系统Avalon、通讯子系统Indigo和程序设计模型WinFX(.NET托管)。去年8月，微软曾透漏这些技术将提供给运行Windows XP SP2级Windows Server 2003 SP1的用户，看上去好像Longhorn不再有什么独特的技术提供开发者了，其实，那不确切。 

首先，通过提供这些技术给现有用户，微软极大的拓展了基于Avalon及Indigo的应用程序和服务的市场。给开发者尝尝鲜以激起他们更大的兴趣，更快接受它，以达到Win32几十年才达到的效果；第二，Longhorn将包括一些XP SP2所没有的独特功能，显得其更具价值。当然，最明显的是Aero用户界面将使高品质视频及3D效果成为PC体验的一部分。 

用户们应该了解，Longhorn将包含全新内核，并且对于传统的16位和32位程序兼容性不会像XP那样好了。对于商业用户，微软相信，Virtual PC 2007将会给他们提供帮助。但是，其也会先发布一个Longhorn兼容性工具给准备迎接改变的用户们。 

8、Longhorn硬件要求 

可能这是很多人都关心的问题，“Longhorn对于硬件有什么要求？”。到现在为止，微软也没有确切的答案。其实微软回答了，我们也不值得相信，因为微软给我们的最低配置通常真的是最低的！ 

首先，微软说2005年购买的大多数适用XP的机器应该都可以运行Longhorn。不过，不意味着能完全体验到Longhorn的特性，估计只能获取像XP一样的Longhorn体验。 

其次，由于Longhorn高级图形技的缘故，你应该具有支持Longhorn显示模型驱动(LDDM)的显卡。2004年中，微软描述过应该具备支持DirectX 9技术的显卡，现在不清楚要求是否会提高。根据我们了解情况，在Longhorn Beta测试阶段，微软会提供清晰的显卡指导方针。 

下面是Longhorn的硬件要求建议配置： 

桌面CPU：3GHz Pentium 4 超线程技术 530以上 或 Xeon 至强带2MB L2缓存 或AMD Athlon 64，Sempron 或 Opteron 100,200,800，单核心或双核心处理器 
移动CPU：1.86 Pentium M 750以上 或 AMD Turion 64 移动技术，移动Sempron，移动版Athlon 64处理器 

内存：512MB以上，所有平台（个人觉得至少1GB） 

9、更新文档 

我们将跟随Paul Thurrott更新最新的Longhorn消息。估计显著的更新会发生在今年的4月和9月，那时WinHEC 2005和PDC 2005召开。
