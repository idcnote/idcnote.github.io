Windows Phone Mango对战Android 2.3 谁更强

内容摘要
卷土重来的新款微软移动设备OS原来只能算作一轮小小的升级，既缺乏企业级安全性保障又没有丰富的应用程序支持；相比之下谷歌的Android智能手机系统则算
文章正文

卷土重来的新款微软移动设备OS原来只能算作一轮小小的升级，既缺乏企业级安全性保障又没有丰富的应用程序支持；相比之下谷歌的Android智能手机系统则算得上更明智的选择。

InfoWorld最近有一篇横评Windows Phone Mango与Android的文章，原文名为《Beauty and the geek: Windows Phone 'Mango' vs. Android》，把 Windows Phone Mango比喻成美女，而Android就是极客。让我们看看最终谁会赢得这场胜利，美女还是极客？

以下为全部译文：

Windows Phone Mango对Android 2.3: 网页与互联网

谷歌拥有HTML 5及其它现代浏览器技术的强力支持，因此其提供的网页浏览器在功能方面具备优势也毫不令人意外。而与谷歌相比，微软在支持新标准方面的脚步似乎过于迟缓。虽然根据HTML 5 测试网站的得分表明，“芒果”在与其竞争对手比起来在HTML 5的兼容性方面取得重大进展，但它仍然落后于其它所有移动版浏览器。

从工作角度来看，Windows Phone与Android浏览器的本质不同在于UI。它们的接口都相当少，一贯的URL地址栏也得以保留; 在Android中可以通过一个图标按钮打开书签页面，而在Windows Phone上则可以通过一个图标按钮刷新网页。它们在实现返回浏览历史记录方面所使用的都是各自系统中标准化的实体后退按钮。如果要在Android系统中实现添加书签、前进、打开新的浏览器窗口或者刷新页面等操作，大家需要使用设备上的实体菜单按钮。而在Windows Phone上，要创建或访问书签(在这里书签被称为‘收藏夹’)以及打开新选项卡(事实上相当于一个新窗口)，我们需要使用更多的图标按钮才能打开菜单选项。Windows Phone 7同样允许用户将某个网页添加到启动屏幕中(类似于iOS系统)；而 Android则不支持此功能。

Windows Phone与Android系统都可以通过电子邮件共享网页，此外Windows Phone还允许我们通过社交网络分享网页。

使用了智能手机上的实体“搜索”按钮来完成浏览器中的网页搜索功能，而Windows Phone则通过其实体“搜索”按钮启动Bing应用程序。不过两者都无法搜索我们的当前页面。

Windows Phone提供了一个.com按钮，当输入网址时可以帮助我们显著地节约时间。另外，当我们点选并长按.com按钮时它还会弹出一个备选后缀名列表，例如.edu以及.org。Android“姜饼”系统则不具备此类功能。

在Windows Phone与Android中，我们可以直接选定网页上的文字及图形，但只有Windows Phone允许大家保存当前选定的内容。

Windows Phone与Android浏览器都提供了控制cookie及访问历史的设定，但与Android不同，“芒果”不具备用来管理其它个人信息，诸如缓存、表彰数据、密码、加载图像、自动填充、欺诈警告及调试等功能的选项。当我们不想访问网站的移动设备优化版本时(这往往会导致信息及服务的缺失)，Windows Phone会告知网站发起访问的设备是台式机而非移动设备，这种良好的能力值得称道。

尽管Android系统中没有预装，但Adobe Flash Player仍然可以在Android Market中免费下载获得。我发现最新的 Flash Player(10.3版本)在视频及基础Flash动画的处理方面表现不俗，大家可以旋转视角并通过热点打开相关内容，而Flash游戏则时灵时不灵。相比之下，Windows Phone 7完全不支持Flash。

优胜者: Android “姜饼”，它的胜出得益于更强大的HTML 5兼容性及更好的个人信息控制。如果对Flash的支持对大家来说很重要，那么Android将是我们惟一的选择。

Windows Phone Mango对Android 2.3: 定位支持

Windows Phone与Android系统都支持GPS定位，并且都能够以基于Wi-Fi信号进行三角定位。它们还具备同等效果的地图应用程序，能够帮助我们找到当前目的地、提供方向指引并以各种方式为我们导航。

Android系统中的导航应用程序提供了Windows Phone(包括iOS)所无法比拟的功能性补充，要达到同样的效果大家恐怕要额外购买单独的导航应用才行。

Windows Phone与Android系统都允许开发人员在其应用程序中添加定位信息，因此定位可以看作是另一项原生功能。这两款操作系统也都允许大家对位置隐私加以控制，但控制水准并不理想：在整台设备上禁用或启用GPS及Wi-Fi定位服务。Windows Phone及Android应用程序会询问是否被允许使用用户当前所处的位置，但是与iOS不同，其中并没有一套核心机制来管理这些位置的使用权限。

优胜者: Android，这次胜利的得来要归功于其自有导航应用程序。如果不考虑这一点，两者基本上旗鼓相当。
