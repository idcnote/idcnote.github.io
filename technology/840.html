虽然Windows 8和的Linux发行版，在设计思想上有较大的区别。但是，最后当然也一样重要的，它们都要以相同的基本原则，面向它们的用户。所以，这两类操作系统设计必然有一些重叠的地方。虽然，微软一直有被指控从开源社区借来一些特性进行模仿，据一些Linux迷说，微软也向一些好的Linux特性靠近。

我一直密切地关注着Windows 8的发展，注意到Windows 8特性上的一些强烈的反冲力。特别是在一些Linux/Windows论坛和在建设中的八个博客，在那里Sinofsky和朋友们写了大量Windows即将到来的叠代。

我对他们指出的一些地方非常得好奇，也很想知道微软是如何把这些特性融合进Windows 8皮肤中的。下面就让我们来看看，Windows 8从Linux社区借鉴了哪些特性。

1.文件拷贝对话

为了创造更高的透明度，微软实施改进了复制、移动、重命名和删除等操作对话，那不仅仅展示了每一步操作的效果，也是一种吞吐量的展现。

哦，这将导致一个开放源社区的大爆发!

一些非常相似的对话已经成为Dolphin和Nautilus文档管理的一部分，文件传输对话也让用户停止操作，在一个窗口中观察多点拷贝工作。我们甚至可以巧妙地捕获带宽用户图，以帮忙展现曾经出现的精彩细节。

微软twist说：当有关于文件操作问题出现时，Windows 8并不只是停止整个过程，并且把错误保持在错误队列中。然而，很明显，微软想好好地看看开源世界在这里的表现。

Linux和Windows 8都没有的是一个排队特征。当然，你可以手动停顿和重新开始拷贝操作。但是，如果是在大量复制任务的情况下，并不能帮到你什么。不过是Linux还是Windows 8的用户们都盼望着解决这个问题已经很久了。

2.ISO文件的挂载

在Windows 8里，微软最后介绍了装配ISO文件。一旦装配，一个新的的驱动器出现在Windows浏览器里，那代表虚拟CD/DVD ROM的存在。虽然这是一个很好的补充，让用户终于摆脱恼人的第三方的工具，如Daemon Tools、Power ISO或者Virtual CloneDrive，无论是Linux还是Mac有这个能力已经有相当一段时间了。

微软twist说：没有Linux distro，ISO挂载和Windows 8一样早。那就需要一些命令行诡计(或者，第三方工具)。感谢所有评论者的诡计，当然，ISO安装对于各种Linux发行版GUI和命令行来说，都是很容易的。

3.Windows To Go

Windows To Go允许(企业)用户创造一个可引导的Windows 8环境通过USB 2.0/3.0闪存。它甚至支持非开源的驱动，使操作系统瞬间冻结，直到你把Windows To Go控制杆拉回去。太牛了!

微软twist说：很明显，在Linux的世界，这样的“生活环境”已有很长一段时间了。但是他们的表现对于一个自然运行的操作系统却显得很不太正常。 自从，微软尝试优化他们的NTFS文件系统，对于这样一个场景。Windows 8甚至可以流畅地运行在USB 2.0上。在测试Windows To Go的时候，我发现无论是引导和全面的速度都远远超过任何我曾经测试过的Linux发行版。

4.Metro用户界面

对于Metro用户界面来说，最初的基本想法出现在五年多以前的媒体中心和Zune硬件上。当你第一次Metro用户界面时，你会发现这是一个非常独特的工作方式和设备。但是，微软不是这个想法的先锋。

各种Linux发行版，如Ubuntu和GNOME桌面环境等，都在微软之前，大检查用户界面，试图找到一个适合“所有界面的UI”的方法。Linux 的用户界面更新是不可否认的，尤其是Ubuntu，被制作成适合于平板电脑的UI。但是，即使是最狂热的Linux用户也承认支持接触，被认为是毫无意义 的事情。

微twist说：微软正以一种非常危险的步骤推动Metro用户界面成为新操纵系统默认的桌面，但是它确实对于触摸和一支钢笔来说更加地舒适。

5.社交整合

Linux发行版，特别是Ubuntu，很长一段时间里，都默认地包括社会媒体整合。“我”菜单，第一次出现在早期Ubuntu 10.04的Alpha版，它允许您更新所有的帐户并在您的桌面上直接获得重要的信息。最后，当微软添加了Tweet@Rama，相片选取和社交名媛应用程序开发者预览。

微twist说：微软差一点就没有赶上这一波社交整合的趋势。

6.本地支持USB 3.0

在他们最初的博客里，解释他们的新本土USB 3.0堆栈，当然，新闻评论的反应是“Linux已经这么做了3年了”。

微软扭：继续往前走，没有什么好看的。USB 3.0装置在Windows 7上运行的很好，也是因为硬件制造商提供了他们自己的驱动，视窗7已从硬件厂商提供自己的司机。微软最后实施了一个行业标准。

7.云整合

不管是Windows 8还是Linux，都可以用云让你同步刷新自己的数据。在Ubuntu 11版本中，它提供了一个5GB空间的免费在线备份服务。如果你想要更多的存储空间，可以选择购买花费每个月2.99美元获得20 GB存储空间。

微软twist说：Windows 8打算紧紧地整合Skydrive的25GB在线存储空间，它不仅用来存储照片和音乐，而且运行托管你的用户账户(包括，个人设置、背景和一些数据)，是你可以在任何地方登陆。

相对地，Ubuntu正在发展它们新的音乐流媒体服务。

8.ReFST

最新消息称ReFST(Resilient File System，弹性文件系统)代号Protogon，是微软下一代的文件系统。它将第一次亮相是在Windows Server 8上，但是客户端采用还在酝酿之中。

ReFST系统本身与ZFS(Z文件系统)和Linux的Btrfs(b树文件系统)，而且当它连接到微软存储空间的时候，也支持读和写快照。为进一步的安全，它也还提供完整checksums和B+树。同时，增加文件、卷、目录大小等，也惊人地类似Btrfs。

微软twist说：我仅仅是想说，微软没有从头开始做事。虽然，我没有深入地研究文件系统驱动，我怀疑微软多年前就开始努力在Btrfs和ZFS中寻找“灵感”，开发一些非常相似的功能。
